# pxd 블로그 개념 위키

LLM이 pxd 블로그 글을 읽고 정리한 개념 페이지 모음.

## UX 이론
- [게슈탈트 법칙과 시각적 통일성](concepts/게슈탈트-법칙과-시각적-통일성.md) — 통일성(Unity)은 디자인 요소들 사이에 시각적·형태적 유사성이 존재해 보는 이가 조화, 일치, 일관성을 느끼는 것
- [밀러의 매직 넘버](concepts/밀러의-매직-넘버.md) — 1956년 조지 밀러가 발표한, 인간의 단기 기억 용량은 7±2개라는 법칙
- [어포던스](concepts/어포던스.md) — 디자이너 사이에서 가장 자주 쓰이는 전문 용어 중 하나이지만, 정작 정의가 혼란스러운 개념
- [피츠의 법칙](concepts/피츠의-법칙.md) — 1954년 폴 피츠가 발표한 운동 행동 모델로, 목표 영역을 클릭하는 데 걸리는 시간 예측
- [힉의 법칙](concepts/힉의-법칙.md) — 선택지 수가 늘어날수록 결정 시간이 증가한다는 법칙

## UX 방법론
- [UX 라이팅](concepts/UX-라이팅.md) — 디지털 서비스 내 모든 텍스트를 사용자 관점에서 설계하는 분야
- [UX 리서치 통계 분석](concepts/UX-리서치-통계-분석.md) — UX 리서치에서 수집한 숫자 지표를 통계적으로 해석하는 방법
- [UX와 UI의 차이](concepts/UX와-UI의-차이.md) — UI는 사람과 정보기기가 접하는 면, UX는 사용자가 제품·서비스를 통해 얻는 총체적 경험
- [디자인 사고](concepts/디자인-사고.md) — IDEO의 팀 브라운이 체계화한 혁신 방법론
- [정보 구조 설계 IA](concepts/정보-구조-설계-IA.md) — 서비스의 전체 구조를 지탱하는 뼈대로서의 정보 구조
- [정보 디자인 원칙](concepts/정보-디자인-원칙.md) — 사용자가 정보를 효과적으로 인지하고 행동할 수 있도록 설계하는 원칙

## 디자인 시스템
- [BX 디자인과 브랜드 자산 운용](concepts/BX-디자인과-브랜드-자산-운용.md) — BX 디자이너는 브랜드의 시각적 언어와 사용자 경험을 통합하는 역할
- [UI 컴포넌트 용어](concepts/UI-컴포넌트-용어.md) — UI 기획자와 디자이너, 개발자가 협업할 때 필요한 공통 언어
- [디자인 시스템](concepts/디자인-시스템.md) — 디자인 원칙, 규격, 재사용 가능한 UI 패턴과 컴포넌트, 코드를 포함하는 종합 세트
- [조형의 요소와 원리](concepts/조형의-요소와-원리.md) — 디자인을 구성하는 재료(요소)와 배합 방법(원리)
- [피그마 프로토타이핑](concepts/피그마-프로토타이핑.md) — 피그마의 프로토타이핑 기능으로 인터랙티브 시뮬레이션 제작

## UI/인터랙션 설계

- [한글 입력 설계](concepts/한글-입력-설계.md) — 자소 빈도 분석에 기반한 한글 자판 설계
