# 윈도우 8 모던 UI

2012년 10월 정식 출시된 **Windows 8**은 기존 Windows 시리즈의 상징이던 시작버튼·시작메뉴를 완전히 없애고, Windows Phone과 동일한 **Modern UI**(당초 'Metro UI'로 불렸으나 상표권 문제로 2012년 8월 개명)를 전면에 내세웠다. 이는 [[스큐어모피즘]]에서 다룬 '메타포 대 메트로' 논쟁의 실제 구현체로, 태블릿 사용을 염두에 둔 좌우 스크롤 전용 시작화면과 Windows Phone 잠금화면과 통일된 인터페이스로 나타났다. 윈도우 키를 누르면 어떤 상황에서든 항상 이 시작화면으로 복귀하도록 설계되어, 윈도우 키가 모바일 기기의 홈버튼과 같은 역할을 맡게 됐다.

Modern UI의 핵심 구성요소는 **Tile**이다. 화려한 아이콘 대신 넓은 사각형에 심플한 흰색 심볼과 글씨로 정보를 표현하며, 새 정보가 도착하면 앱 실행 없이도 실시간으로 내용이 갱신되는 **Live Tile**로 확장된다. **Charm bar**는 화면 우측 상단·하단(터치는 오른쪽에서 왼쪽 스와이프)으로 호출하는 검색·공유·시작·장치·설정 5가지 기능 모음인데, 각 기능이 현재 실행 중인 앱 맥락에 따라 다르게 동작하는 **contract** 방식으로 설계돼, 앱마다 제각각이던 설정 화면을 하나의 통일된 인터페이스로 흡수하려 했다. 여러 앱을 썸네일로 전환하는 **Switching pane**, 화면을 좌우로 분할해 두 앱을 동시에 조작하는 **Snap**, 핀치 줌으로 전체 콘텐츠를 조망한 뒤 원하는 항목으로 빠르게 드릴다운하는 **Semantic Zoom**('Fast and Fluid' 슬로건을 구현하는 핵심 내비게이션)도 이 시기 새롭게 등장한 상호작용 방식이다. 다만 Semantic Zoom의 최소화 버튼처럼 작은 대상을 마우스로 정확히 누르기 어려운 지점은 [[피츠의 법칙]] 관점에서 여전히 아쉬운 부분으로 지적됐다.

MS는 가이드라인을 통해 **Touch-First** 설계를 권고하면서도 손글씨 입력기·화상키보드를 함께 제공해 기존 마우스·키보드 사용자를 배려했다. 그러나 근본적인 한계는 하나의 OS 안에 이질적인 두 UI—Modern UI와 전통적 Desktop UI—가 공존한다는 데 있었다. 시작화면의 Internet Explorer와 데스크톱의 Internet Explorer가 서로 호환되지 않는 별개 프로그램이었고, 데스크톱에서는 Charm bar의 공유 기능조차 쓸 수 없는 등 두 공간의 경험이 어긋났다. 윈도우 키를 누를 때마다 Modern UI 시작화면으로 튕겨나가는 구조 때문에, 사용자는 마치 서로 다른 두 개의 OS를 오가는 듯한 이질감을 느끼게 됐다. 이 문제는 데스크톱을 아예 포함하지 않는 태블릿 전용 **Windows RT**와, 기존 PC 호환을 위해 두 UI를 함께 담은 **Windows 8 RTM**으로 제품 라인이 갈리는 배경이기도 했다.

결과적으로 Modern UI는 터치 조작의 편의성, 빠른 내비게이션, 앱을 실행하지 않고도 최신 정보를 확인할 수 있는 Live Tile 등에서 혁신적이고 매력적인 시도였지만, Modern UI와 Desktop UI 사이의 괴리는 과도기적 산물로서 뚜렷한 한계로 남았다.

## 핵심 내용
- **Modern UI**(구 Metro UI): Windows Phone과 통일된 시작화면·잠금화면, 시작버튼 폐지
- **Tile / Live Tile**: 심플한 흰색 심볼 타일, 앱 미실행 상태에서도 실시간 정보 갱신
- **Charm bar**: 검색·공유·시작·장치·설정 5기능을 실행 중인 앱 맥락(contract)에 따라 통일 제공
- **Switching pane / Snap**: 앱 전환 및 화면 좌우 분할 멀티태스킹
- **Semantic Zoom**: 핀치 줌으로 전체 조망 후 드릴다운하는 'Fast and Fluid' 내비게이션
- **Touch-First** 가이드라인 + 손글씨 입력기·화상키보드로 마우스/키보드 병행 지원
- 근본 한계: **Modern UI ↔ Desktop UI 이질성** — 같은 이름 앱(IE 등)이 서로 호환되지 않는 별개 프로그램으로 존재
- 제품 라인 분화: 데스크톱 없는 **Windows RT** vs 두 UI를 모두 담은 **Windows 8 RTM**
