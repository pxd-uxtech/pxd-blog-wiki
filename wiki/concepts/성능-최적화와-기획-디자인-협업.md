# 성능 최적화와 기획 디자인 협업

**성능 최적화**는 흔히 개발 단계의 문제로 여겨지지만, 실제로는 기획서의 데이터 갱신 주기나 디자인의 이미지·애니메이션 사양이 확정되는 순간부터 성능의 상한선이 정해진다. pxd UX Engineer **Seulbi Lee**는 여러 프로젝트 경험을 바탕으로, 기술적 구현은 개발자의 몫이 맞지만 정말 좋은 성능은 **기획과 디자인 단계부터 함께 고민할 때** 나온다고 말한다.

기획 단계에서 놓치기 쉬운 성능 포인트로 세 가지 사례가 제시된다. 한 번에 100~500개씩 데이터를 가져오는 무한스크롤 설계는 DOM 노드 누적에 따른 메모리 증가와 API 응답 지연을 유발하며, **가상 스크롤링**으로 개선할 수 있다. "모든 데이터를 1분마다 실시간 갱신"하는 요구는 다수의 API 호출로 인한 서버 부하와 불필요한 리렌더링을 낳으므로, 사용자가 실제로 그 갱신 주기를 필요로 하는지부터 검토해야 한다. 너비 50px에 불과한 차트 영역에 분 단위 데이터를 그리도록 요구하는 경우처럼, 실제 표현 가능한 해상도를 넘어서는 데이터 세분화 요구도 흔한 함정이다.

디자인 단계의 문제는 더 직관적이다. 4K 해상도의 5MB 히어로 이미지, 반응형 브레이크포인트마다 다른 이미지, 페이지당 20개 이상의 고해상도 이미지, 5~6개의 웹폰트, 복잡한 그라데이션·그림자 조합은 로딩 시간 8초, Lighthouse 성능 점수 30점이라는 결과로 이어진다. `transition: all`, `backdrop-filter: blur()`, hover 시 `transform + scale + rotate`를 동시에 적용하는 화려한 카드 애니메이션도 60fps를 보장하기 어렵고, GPU 가속을 받는 transform·opacity 위주로 설계해야 리페인트·리플로우 비용을 줄일 수 있다(관련 원리는 [[브라우저 렌더링 파이프라인]] 참고).

이런 요구가 반복되는 근본 원인은 기술적 스킬 부족이 아니라 **정보 공유 부족**이다. 개발자가 성능 우려를 제기하면 "다른 사이트도 이렇게 한다", "이미 컨펌이 끝났다"는 답이 돌아오는 경우가 많고, 이 과정이 누적되면 성능이 개발자 개인의 스킬 문제로 축소된다. 기획자·디자이너에게도 더 풍부한 인터랙션과 차별화된 시각적 임팩트를 원하는 합리적 이유가 있으므로, 문제 해결의 핵심은 비난이 아니라 **각 요구가 성능에 미치는 영향에 대한 정보를 공유**하는 것이다.

해결책은 단계별 가이드라인과 협업 프로세스로 구체화된다. 기획 단계에서는 페이지네이션·가상 스크롤링 도입, 부분 업데이트, 사용자 행동 패턴에 근거한 갱신 주기 설정, 필수 기능과 점진적 로딩 기능의 계층화가 제시된다. 디자인 단계에서는 히어로 이미지 1920px·100KB 이하, 썸네일 300px·20KB 이하, WebP 대체안 준비, 웹폰트 2~3개 이하, `transform`·`opacity` 위주에 0.3초 이하 지속시간, 동시 애니메이션 5개 이하 같은 구체적 수치 기준을 제안한다. 협업 프로세스 측면에서는 기획·디자인 단계별 성능 체크리스트 도입, 스프린트 시작 전 성능 임팩트 검토, 프로토타입 단계 성능 측정, 배포 전 [[코어 웹 바이탈과 SEO|Core Web Vitals]] 점검을 정기화하는 문화가 강조된다.

## 핵심 내용
- 성능은 개발 단계가 아니라 **기획·디자인 단계에서부터 결정**되는 문제
- 기획 함정: 대량 무한스크롤(→가상 스크롤링), 과도한 실시간 갱신 주기, 뷰포트 대비 과도한 데이터 세분화
- 디자인 함정: 5MB급 히어로 이미지, 웹폰트 남용, 복잡한 그림자·그라데이션·`backdrop-filter` → 로딩 8초·Lighthouse 30점
- 애니메이션은 `transform`·`opacity` 위주로 설계해야 GPU 가속을 받아 60fps 유지에 유리
- "다른 사이트도 이렇게 한다"는 식의 소통은 성능을 개발자 개인 스킬 문제로 축소시킴 — **정보 공유 부족**이 근본 원인
- 이미지 총용량 페이지당 1MB 이하, 웹폰트 3개 이하 등 구체적 수치 기준을 팀 공통 가이드라인으로 문서화 권장
- 스프린트 시작 전·프로토타입 단계·배포 전 등 정기적인 성능 리뷰 문화 정착이 핵심 해결책
