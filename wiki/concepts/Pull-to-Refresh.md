# Pull to Refresh

**Pull to Refresh(당겨서 새로고침)**는 모바일 앱에서 콘텐츠를 갱신하기 위해 화면을 아래로 당기는 제스처 인터페이스 패턴이다. 실시간 자동 갱신은 서버와 네트워크 자원을 많이 소모하므로, 사용자가 필요할 때 직접 새로고침을 요청하는 방식이 효율적이다. 전통적인 버튼 방식에서 발전한 이 패턴은 지금은 모바일 UI의 표준으로 자리 잡았다.

이 UI는 개발자 **로렌 브릭터(Loren Brichter)**가 2009년 트위터 클라이언트 **Tweetie**에서 처음 도입했다. 2010년 트위터가 Tweetie를 인수하면서 **Pull to Refresh UI 특허**를 미국 특허청에 출원했고, 이후 발명가 혁신 특허 계약(**Innovator's Patent Agreement**)을 통해 특허를 방어 목적으로만 사용하겠다고 선언해 널리 퍼질 수 있었다.

인터랙션은 세 단계로 구성된다. 첫째, 당기기(Pull): 화면을 아래로 스크롤해 임계점에 도달하면 새로고침이 시작됨을 알린다. 둘째, 해제(Release): 손을 떼면 갱신이 실행된다. 셋째, 갱신(Refresh): 로딩 인디케이터가 표시되며 콘텐츠가 업데이트된다. **피츠의 법칙** 관점에서 이 패턴은 화면 상단 버튼을 눌러야 했던 기존 방식보다 손 이동 거리를 줄이고, 엄지로 조작하는 한 손 사용에도 유리하다.

25개 앱을 분석한 연구에 따르면 구현 방식은 크게 세 유형으로 나뉜다. **기본형**은 원래 요소(텍스트·화살표·진행 인디케이터)를 그대로 유지하는 방식, **변형형**은 일부 요소를 변경하는 방식, **혁신형**은 완전히 새로운 시각적 접근을 채택하는 방식이다. iOS Mail, Twitter Music 등이 독창적인 애니메이션으로 혁신형의 대표 사례를 보여준다.

2013년과 2018년, 5년의 간격을 두고 동일한 18개 앱(Facebook, Twitter, Youtube, 쿠팡, Pocket, Fancy, 카카오스토리, Pinterest, 싸이월드, 네이트온, 비트윈, Vimeo, iOS Mail, Podcast, Gmail, Google+, Tumblr, Path)의 Pull to Refresh UI를 재비교한 후속 조사에서는 뚜렷한 **간소화(simplification)** 경향이 확인됐다. 화살표 아이콘·텍스트 정보·컬러풀한 애니메이션 등 초기의 화려한 구성 요소는 대부분 사라지고, 스피너 형태의 인디케이터만 잠깐 회전하다 사라지는 형태로 수렴했다. 이 변화의 배경으로 세 가지 요인이 꼽힌다. 첫째, **학습 완료**: 당겨서 새로고침이라는 제스처는 핀치 줌처럼 이미 모든 사용자가 알고 있어 별도의 설명·학습 요소가 불필요해졌다. 둘째, **네트워크 속도 향상**: 데이터 로딩 시간이 짧아지면서 새로고침 구간에 복잡한 정보를 담을 시간적 여유 자체가 줄었다. 셋째, **주의 분산 방지**: 좁은 모바일 화면에서 콘텐츠 몰입을 방해하지 않도록 굳이 존재감을 드러내지 않는 방향으로 최소화됐다. 반면 쿠팡·옥션·Amazon 등 **이커머스** 서비스는 여전히 브랜드 로고·상품 카테고리·원형 아이콘 등을 넣은 상대적으로 화려한 새로고침 UI를 유지하거나, 쿠팡처럼 당기는 대신 무한 스크롤로 콘텐츠를 선반영하는 방식으로 전환하는 등 콘텐츠 앱과 다른 궤적을 보였다. 5년 사이 Pose·Vyclone·Readability·Intonow·Vine·Dashboard·Twitter Music 등 비교 대상 앱 중 다수 서비스가 종료되기도 했다.

## 핵심 내용
- 최초 도입: 2009년 **Loren Brichter**의 Tweetie, 이후 트위터가 인수 후 특허 출원
- 인터랙션 3단계: 당기기(Pull) → 해제(Release) → 갱신(Refresh)
- **피츠의 법칙** 적용: 상단 버튼 방식 대비 손 이동 거리 감소, 한 손 조작 편의 향상
- 구현 유형: 기본형 / 변형형 / 혁신형으로 분류
- 특허는 방어 목적에만 사용한다는 Innovator's Patent Agreement로 업계 전파 가능
- 2013→2018년 재조사 결과: 학습 완료·네트워크 속도 향상·주의 분산 방지 세 요인으로 UI 요소가 대거 간소화(스피너 인디케이터 중심)
- 이커머스 앱은 여전히 화려한 피드백을 유지하거나 무한 스크롤 방식으로 전환, 콘텐츠 앱과 다른 흐름을 보임
