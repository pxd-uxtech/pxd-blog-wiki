# 약관 동의 UI

약관 동의 화면은 금융·커머스·앱 서비스에서 반드시 필요한 UI이지만, 잘못 설계하면 사용자에게 혼란과 불편을 주는 대표적인 마찰 지점이 된다. 신한카드 Fan 앱카드 사례 분석을 통해 약관 동의 UI의 흔한 문제점과 개선 방향을 살펴볼 수 있다.

**입력의 불편함**: 체크박스가 과도하게 많고, 대카테고리 약관에 동의하면 하위 소약관이 갑자기 펼쳐지는 방식은 사용자에게 혼란을 준다. 카카오 뱅크처럼 '전체 동의' 버튼을 주 행동으로 두고, 선택 약관은 별도로 표시하는 방식이 더 직관적이다.

**레이블의 문제**: [선택] 카테고리 하위에 [필수] 약관이 섞여 있으면 사용자가 무엇이 필수인지 혼란스럽다. 필수 약관과 선택 약관을 명확히 분리해 표시해야 하며, [선택] 항목인데도 선택하지 않으면 다음 단계로 진행되지 않는 모순도 피해야 한다.

**항목 구별의 문제**: 필수·선택 약관이 시각적으로 구분되지 않으면 사용자가 전체 구조를 파악하기 어렵다. 약관명의 띄어쓰기 불일치 같은 사소한 비일관성도 서비스 신뢰도에 영향을 미친다.

좋은 약관 동의 UI는 사용자가 구조를 한눈에 파악하고, 필수와 선택을 직관적으로 구분하며, 최소한의 입력으로 완료할 수 있어야 한다.

## 핵심 내용
- 과도한 체크박스와 갑작스러운 약관 펼침은 주요 UX 문제
- 필수 약관과 선택 약관을 시각적으로 명확히 구분해야 함
- '전체 동의' 버튼을 주 행동으로, 선택 약관은 부차적으로 배치
- [선택]이라고 표시된 항목이 실제로는 필수인 모순 설계 금지
- 약관명 등 텍스트 레이블의 일관성도 신뢰도에 영향
