# 네비게이션 레이블링

**네비게이션 레이블링**은 메뉴, 카테고리, 링크 등 사용자가 탐색에 사용하는 모든 텍스트를 설계하는 작업이다. UI 기획에서 가장 어려운 과제 중 하나로, 잘못된 레이블은 사용자가 원하는 정보를 찾지 못하게 만든다. *Designing Web Navigation*에서 제시한 6가지 원칙이 실무 기준으로 널리 활용된다.

**A. 사용자 언어로 말하기**: 레이블은 서비스 제공자가 아닌 사용자의 언어로 작성해야 한다. 회사 내부 전문 용어나 기술 용어는 사용자에게 혼란을 준다. 지나치게 영리하거나 재치 있는 표현도 피해야 한다. 약자(FAQ, PDF, RSS 등)는 모든 사용자가 이해하지 못할 수 있으므로 신중하게 사용한다. 서비스 성격에 맞는 **보이스 톤** 유지도 중요하다—은행 사이트와 10대 음악 사이트의 레이블 톤은 달라야 한다.

**B. 설명적인 레이블**: 레이블은 해당 콘텐츠의 단서를 제공해야 한다. 넓고 모호한 레이블을 써야 하는 경우라면, 어떤 방식으로든 의미를 한정하는 노력이 필요하다.

**C. 상호 배타적인 레이블**: 메뉴 그룹 내 각 레이블의 의미가 겹쳐서는 안 된다. 인접한 레이블이 서로 의미에 영향을 주기 때문에, 레이블 간 경계를 명확히 구분해야 한다.

**D. 명확한 초점을 나타내는 레이블**: 초점이 맞춰진 레이블은 예측 가능성을 높이고 사용자의 탐색 자신감을 증대시킨다. "동물"보다 "애완동물"이 더 좋은 예이며, "고양이과 & 개과"처럼 지나치게 상세하면 다른 대상이 분류에서 빠지게 된다.

**E. 일관된 레이블**: 일관성은 모호성을 제거한다. 네 가지 차원의 일관성이 있다. ① **정밀도** — 동일 위계의 레이블은 동일한 콘텐츠 폭을 가져야 한다. ② **구문론** — 모든 레이블이 유사한 문법 구조를 따라야 한다(예: 명사형 통일). ③ **표현** — 서체, 크기, 스타일을 일관되게 적용한다. ④ **사용** — 동일한 항목은 여러 위치에서 동일한 레이블을 사용하며, 인쇄물 등 다른 채널에서도 동일하게 유지한다.

**F. 레이블의 길이**: 긴 레이블은 사용자가 찾는 단서를 포함할 가능성이 높지만, 13단어 이상의 레이블은 성능이 크게 떨어진다는 연구(Spool)가 있다. 짧은 레이블을 쓰되 옆에 설명문을 덧붙이는 방식이 명확성과 간결성을 동시에 확보하는 방법이다.

**실패 사례: 미투데이(me2Day) 친구찾기**. 원칙 위반이 사용자 혼란으로 직결된 실제 사례다. 미투데이는 '미투데이'와 '마이미투'라는 완전히 다른 두 화면을 탭할 때마다 오가며, 상단 메뉴 구성 전체가 그때그때 다른 모습으로 바뀌어 사용자가 지금 어느 화면에 있는지조차 헷갈리게 만든다. 더 큰 문제는 **'친구 찾기'라는 동일한 레이블이 서로 다른 위치에서 서로 다른 기능을 가리킨다**는 점이다. '내 친구들' 탭 안의 '친구 찾기'는 이미 등록된 친구들 중에서만 찾는 기능인 반면, 실제로 새로운 미투데이 회원을 찾는 기능은 '내 친구들' 하단의 별도 '친구 찾기'에 있다. 상호 배타적이어야 할 레이블이 위치만 다를 뿐 같은 이름으로 중복 사용되면서, 사용자는 여러 번의 시행착오를 거치고서야 원하는 기능을 찾을 수 있었다. 이는 원칙 C(상호 배타적인 레이블)와 E(일관된 레이블, 특히 '사용'의 일관성)를 동시에 위반한 사례다.

## 핵심 내용
- 레이블은 **사용자 언어** 기준으로 작성—전문 용어, 기술 용어, 약자, 과도한 위트 지양
- 레이블은 콘텐츠의 단서를 제공해야 하며 **모호함을 피할 것**
- 메뉴 내 레이블은 **상호 배타적**으로 설계해 의미가 겹치지 않도록
- 동일 위계 레이블은 **초점과 범위가 균일**해야 예측 가능성 향상
- **일관성** 4가지: 정밀도, 구문론, 표현, 사용
- 레이블 길이는 13단어 미만 권장; 짧은 레이블 + 부연 설명 조합이 효과적
- 실패 사례(미투데이): 동일한 '친구 찾기' 레이블이 서로 다른 뎁스에서 다른 기능을 가리켜 상호 배타성·일관성 원칙을 모두 위반
