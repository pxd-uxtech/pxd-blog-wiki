# 대중교통 앱 UX

대중교통 앱의 핵심 사용 맥락은 크게 두 가지로 나뉜다. **익숙한 경로(출퇴근 등 반복 이동)**와 **낯선 경로(새로운 목적지 탐색)**이다. 이 두 시나리오에서 사용자의 정보 필요와 앱 의존도가 크게 다르기 때문에, 동일한 UI가 두 맥락 모두를 충족하기 어렵다. 카카오지하철 앱 분석에서 드러난 것처럼, 익숙한 경로에서는 즐겨찾기·알람 기능으로 충분하지만, 낯선 경로에서는 목적지 주변 역 파악, 출구 번호, 도보 방향 정보까지 필요하다.

**현재 위치 자동 감지** 기능(가장 가까운 역 자동 지정)은 사용자의 출발역 입력 부담을 크게 줄이는 핵심 편의 기능이다. 다만 이러한 기능이 존재하더라도 **아이콘이 의미를 직관적으로 전달하지 못하면** 사용자가 기능을 발견하지 못한다. 도착 예정 시간 공유(카카오톡 연동), 환승·하차 전 알람 등은 실제 이동 맥락에서 유용하게 쓰이는 기능이다.

앱 내 **정보 단절(Context Break)** 문제도 중요한 UX 이슈다. 목적지가 어느 역 근처인지 파악하려면 지도 앱으로 이동해야 하고, 하차 후 몇 번 출구로 나가야 할지는 다시 지도 앱을 확인해야 한다. 경로 이탈 감지(내릴 역을 지나쳤을 때 알림)도 현재 제공되지 않는다. 이처럼 단일 서비스 내에서 해결되지 않는 정보 공백은 사용자가 앱 간 전환과 스크린샷 등의 우회 행동을 하도록 만든다. **서비스 간 유기적 연결(카카오지하철 ↔ 카카오맵)**이 개선 방향으로 제시된다.

환승 여유 시간 표시도 개선 여지가 있다. 단순히 "2분"이라는 수치를 제시하는 대신, **여유 있음/보통/촉박** 같은 상대적 맥락 레이블을 제공하면 사용자가 이동 속도를 조절하는 데 유용한 정보가 된다.

## 핵심 내용
- 대중교통 앱은 익숙한 경로와 낯선 경로의 두 맥락을 구별하여 설계해야 한다
- 유용한 기능이라도 아이콘이 직관적이지 않으면 **발견 가능성(Discoverability)**이 떨어진다
- 하차 후 경로 안내(출구 → 도보) 정보가 누락되면 사용자는 앱 간 전환으로 우회한다
- 숫자 기반 시간 정보보다 상대적 맥락 레이블이 행동 결정에 더 유용한 경우가 있다
- 경로 이탈 감지는 사용자가 스스로 오류를 발견하는 지연을 줄이는 안전망 기능이다
