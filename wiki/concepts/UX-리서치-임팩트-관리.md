# UX 리서치 임팩트 관리

UX 리서치의 가치는 인사이트를 생산하는 것에서 끝나지 않는다. 리서치가 실제로 조직에 미친 영향을 체계적으로 추적하고 관리하는 것이 **임팩트 관리**다. 임팩트란 "한 사람, 사물, 행동이 다른 것에 미치는 영향 또는 효과"(Julie Zhuo)이며, 두 가지 조건이 충족될 때 의미를 갖는다: 측정 가능한 구체적 결과여야 하고, 수혜자가 특정되어야 한다.

pxd UX 리서치 팀은 업무를 **기초 리서치**(미래 사용자 시나리오 생성)와 **평가 리서치**(제품 개발 지원)로 구분한다. 기초 리서치는 임팩트가 장기간에 걸쳐 나타나 측정이 어렵고, 평가 리서치는 비교적 단기에 측정 가능한 결과를 낳는다.

구글 리서처 **타오 동(Tao Dong)**의 3단계 리서치 임팩트 모델이 자주 참조된다. **Level 1**은 특정 질문에 답하는 리서치로, 즉각적인 임팩트와 높은 연구자 관여도를 가진다. **Level 2**는 주요 의사결정을 지원하는 리서치로, 중간 수준의 시간과 관여도가 소요된다. **Level 3**은 사용자 가치를 창출하는 리서치로, 수개월~수년의 장기 임팩트를 가지며 연구자의 직접 관여도는 낮다. 기초 리서치는 주로 Level 3에 해당한다.

임팩트 관리의 실행 절차는 네 단계다. 1) 리서치 유형(Tao Dong 기준) 정의, 2) 측정 방법(데이터 비교, 설문, 인터뷰 등) 사전 설계, 3) 리서치 완료 후 임팩트 데이터 수집, 4) **임팩트 트래커** 유지·보수. 추적 항목에는 리서치 이름·방법·유형·이해관계자(리서치 관련 항목), 임팩트 유형·측정 방법·이해관계자 피드백·임팩트 범위(임팩트 관련 항목)가 포함된다.

## 핵심 내용
- UX 리서치 임팩트는 '측정 가능한 결과'와 '특정 수혜자'가 있을 때 의미를 가짐
- Tao Dong의 3레벨 모델: Level 1(즉각적 질문 답변) → Level 3(장기적 사용자 가치)
- 기초 리서치일수록 임팩트가 장기적이라 측정이 어려우므로 사전에 측정 방법을 설계해야 함
- 임팩트 트래커: 리서치 정보 + 임팩트 정보를 조직적으로 문서화·관리
- 임팩트 범위는 팀 → 조직 → 조직 외부 3단계로 구분
